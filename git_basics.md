### Erkläre ebenfalls, warum bei einem Feuer zuerst Git Commit – Git Push und dann erst die Flucht erfolgen sollte?

##### Warum Git ind der Softwareentwicklung?
Es ist wichtig damit die Arbeitsstände gesichert werden, dies ist im git einfach da man diese nachvollziehen kann da Commit nachriten verlagt werden welche man schreiben muss, somit schützt man den Code vor Verlust. 
Man kann Fehler rückgängig machen indem man in cen commits naschauen kann wann der code welchen inhalt hatte und diesen wieder in dieser Version öffnen.
Somit kann man mit mehrern Leuten am gleichen Code arbeiten und weiss wer welche änderung vorgenommen hat.

##### Warum erst Commit - Push - exit building?
Es geht darum das der Code gesichtert werden muss, dh. was nicht gepusht ist, existiert nur auf dem eigenen Computer. 
Mit Push ist die Arbeit extern also im Git Repository gesichert und geht nicht verloren.
